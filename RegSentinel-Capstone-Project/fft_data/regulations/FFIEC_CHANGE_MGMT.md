# Change Management Controls

**Citation:** FFIEC IT Examination Handbook — Operations  
**Authority:** FFIEC  

## Summary

All changes to production systems supporting financial reporting and customer data must follow a documented change process.

## Key Requirements

- Associate every change with an approved ticket.
- Test changes in a non-production environment before deployment.
- Require authorization by personnel independent of the requester.
- Conduct post-implementation review of emergency changes within 5 business days.
- Restrict changes outside the approved change window without pre-approval.

## Detection Indicators

- Configuration change with no associated ticket.
- After-hours change to a critical system without documented pre-approval.
- Developer deploying directly to production without independent review.

## Enforcement

Examination findings (MRA/MRIA); elevated IT risk rating; remediation mandates.
